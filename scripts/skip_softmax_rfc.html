Motivation

Why now, and what we're proposing

  • Video DiT inference is attention-bound at long context (Wan 2.2: 81 frames × 720p → >75k tokens).
  • BLASST Skip-Softmax (MLSys'26 best paper) skips negligible attention blocks inside the kernel — no training, no weight change.
  • It composes with SAGE (FP8 attention) and ModelOpt GEMM quant. On Wan 2.2 / B200: up to 1.40× E2E.
  • Blackwell trtllm-gen kernels are already exposed through FlashInfer — kernel parity vs TRT-LLM RC20 validated on B300.

Background · the algorithm

BLASST: skip a block when it can't matter

Riding FlashAttention's online softmax, per key-block j:

m̃ⱼ = rowmax(QKⱼT)          # block max
m  = max(m, m̃ⱼ)             # running max
if  m̃ⱼ − m < ln(λ):  continue   # SKIP block
P  = exp(QKⱼT − m); O += P·Vⱼ  # else accumulate
  • Skip test uses the running max → monotone, so a skip is never retroactively wrong (no rescale needed).
  • Skips the P·V GEMM + V-load; still pays QKT → kernel speedup capped ~2×.
  • Threshold λ = a/L (context-length normalized); a is calibrated, not analytic.

Kernel speedup ≈ 1 + 0.9·sparsity (B200)

SparsitySpeedup
0%1.00×
50%~1.30×
80%~1.60×
~100%~1.85×

BF16 gains more from skipping than FP8 (the un-skippable QKT is a bigger fraction under FP8). Low sparsity → overhead can exceed the win.

Decision: expose a friendly diffusion-side name trtllm that resolves to FlashInfer's trtllm-gen path, with sage / skip_softmax as features under it.

API design · principle

Split the kernel from the DiT policy

BLASST is the kernel; the timestep gate is integration. Keep them in different layers so the backend stays a pure adapter.

Pipeline / runtime (owns the DiT policy)

denoise step + total steps→ gate: dense if t > disabled_until_timestep→ factor = θ(sparsity, L) · L→ md.skip_softmax_factor

Backend (pure kernel adapter — knows nothing about diffusion)

bf16 q,k,v→ SAGE quant → fp8 + block scales→ pack ragged (cu_seqlens)→ trtllm_ragged_attention_deepseek(..., skip_factor)→ bf16 out

API design · user surface

One backend flag + two dials users understand

Deploy YAML (per DiT stage)

stages:
  - stage_id: 0
    attention_backend: trtllm      # → flashinfer trtllm-gen
    trtllm:
      sage: true                   # FP8-SAGE attn quant
      skip_softmax:
        sparsity: 0.65             # speed↔fidelity dial
        disabled_until_timestep: 0.86  # fidelity gate (t:1→0)
        calibration: skip_calib.json   # ModelOpt output

CLI (mirrors upstream --attention-config)

vllm serve my/dit --omni \
  --attention-backend trtllm \
  --attention-config '{"sage":true,
    "skip_softmax":{"sparsity":0.65}}'

The only two concepts a user needs

sparsity (0 → ~0.9): speed↔fidelity. 0 = dense.

disabled_until_timestep (0–1): keep early denoise steps dense — protects visual fidelity.

Everything hidden: FP8/SAGE scales, block layouts, λ=a/L, calibration lookup.

Per-request override

{"prompt":"...", "num_inference_steps":50,
 "extra_body":{"skip_softmax_sparsity":0.6}}

sparsity:0 at request time = force dense (a "high quality" toggle).

API design · schema & safety

Config schema + graceful fallback

@dataclass
class SkipSoftmaxConfig:
    sparsity: float | None = None       # user dial
    calibration: str | None = None      # ModelOpt sparsity↔θ
    disabled_until_timestep: float = 0  # fidelity gate (t:1→0)
    # advanced (usually from calibration):
    theta: float | None = None

@dataclass
class TrtllmAttnConfig:
    sage: bool = True
    sage_block: tuple = (1,4,0,1)        # (Q,K,P,V) Blackwell
    quant_dtype = torch.float8_e4m3fn
    skip_softmax: SkipSoftmaxConfig | None = None

Selector: only run the validated envelope

  • SM100+ (B200 / B300); head_dim = 128; dense, non-causal; MHA (q_heads = kv_heads).
  • Outside it → auto-fall-back to flash-attn with a one-line log. No silent no-ops, no crashes.
  • No calibration and no theta → skip disabled (degrade to dense, never guess).
  • use_trtllm=false / sparsity:0 = hard off switch.
Schema & fallbackNVIDIA
ModelOpt in the loop

Offline once; two jobs, not one

Job 1 · GEMM quant

ModelOpt quantizes linear weights → FP8 / NVFP4, scales baked into the checkpoint.

Loaded by the existing ModelOpt adapter (#5076 / #5087).

Job 2 · Skip calibration

Sweeps thresholds → fits sparsity ↔ θ map + recommends disabled_until_timestep. No weight change.

→ the skip_calib.json the YAML points at.

NOT ModelOpt · SAGE

FP8-SAGE attention scales are dynamic / runtime, per-block — computed each forward. No calibration.

Test plan · correctness

Kernel parity + unit + fallback

Kernel parity (already have the harness)

  • FI trtllm-gen vs pinned TRT-LLM RC20 runner, byte-identical FP8 inputs.
  • Gate: rel-L2 ≤ 2e-3, cos ≥ 0.999, Skip-effect delta ≤ max(2e-4, 0.05·R_ref).
  • Partial-K tails (S257/258/259) + mod-4 cumulative offsets.
  • Result on B300: max rel-L2 1.36e-5 ✓

Unit / regression (in-tree)

  • Reject negative / NaN / inf skip factors.
  • None == 0.0 disable equivalence (≤1e-6).
  • Active factor materially changes output; still finite.
  • Selector: unsupported SM / head_dim / causal → falls back to flash-attn, no exception.
  • Uncalibrated model → skip disabled (dense), not garbage.

Metrics & gates

What we measure, and the bars to pass

Performance

  • E2E latency & speedup vs BF16 compiled-dense (common baseline).
  • Attention kernel time + achieved sparsity (does θ realize target).
  • Peak VRAM (must stay within the KV budget, e.g. ~13 GiB).
  • Gate: FI vs TRT median ≤ 1.02×, p95 ≤ 1.05×.

Fidelity

  • LPIPS-low5 & PSNR vs the model's dense reference.
  • Video: VBench (subject/background consistency, motion, aesthetics).
  • Report the Pareto; pick the default at the knee (quality-safe ≈ 1.25×).
  • Gate: chosen default LPIPS within model-specific budget of dense.

CI: kernel-parity + unit on every PR (L1/L2); E2E model sweeps as nightly (L4) on a Blackwell runner — no perf claims from a contended GPU.

Rollout & open questions

Phased, and honest about limits

Phases

  • P0 — trtllm backend + selector gating + flash-attn fallback (no skip yet).
  • P1 — SAGE FP8 attention path + kernel-parity CI.
  • P2 — Skip-Softmax + skip_calib.json + timestep gate.
  • P3 — per-model calibration + defaults + docs (Wan / Hunyuan / Cosmos 3).

Risks / open

  • Blackwell-only (SM100+); everything else falls back — that's fine, but limits reach.
  • On-the-fly SAGE quant ≠ the validated exact-input path → needs its own E2E quality gate.
  • Per-model calibration artifact ownership + versioning.
  • Ragged packing overhead each step; Skip is the smallest win → opt-in per model, not default-on.

The ask

Sign-off to build P0–P1

Register trtllm as a diffusion attention backend (FlashInfer trtllm-gen), with sage + skip_softmax as composable features, hardware-gated with graceful fallback — then land SAGE + kernel-parity CI. Skip-Softmax + per-model calibration follow in P2–P3.

Decision points: backend naming (trtllm vs flashinfer+config) · calibration artifact ownership · which model leads the E2E bring-up (proposal: Wan 2.2).
